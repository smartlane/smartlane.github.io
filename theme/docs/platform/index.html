SMARTLANE Platform

Introduction

This page gives a technical overview of the SMARTLANE platform. The goal is to provide an explanation of what the platform can do and can facilitate in technical terms for customers wishing to integrate Smartlane's services into their business.

Data gathering and archiving

The lower section of the graphic shows various methods of bringing data into the platform. Data can be regularly pulled from external services either to ascertain the current status or to accumulate (archive) data for later analysis. Triggers can also be used to provide initial processing of such data, for example to detect changes and only save those for later use as events. You can also push data to the platform using our HTTP API which is convenient for devices or apps wanting to send data to the platform. For large amounts of frequently changing data, or for synchronization with your own databases, we can harvest data using streaming. Get in touch to find out what is the most suitable approach for your system.

Data and App management

We use an extended version of the CKAN Data cataloging software for providing you with an overview of what data is in the platform and what is being done with it. Customers receive a login for either the shared smartlane.io platform or a private, e.g. yourbusinessnamehere.smartlane.io, instance. Here you can manage data uploads, scheduled data pulls from APIs, connected apps and so on.

Querying

Apps built on the SMARTLANE platform can take advantage of high performance querying of multiple, integrated data sources thanks to powerful geospatial and temporal querying. Our app development usually takes place in JavaScript, but this querying can take place using any programming language that supports HTTP requests.

Mobility Intelligence Engine

Utilizing highly performant parallel and in-memory data processing techniques, SMARTLANE provides an engine for developing powerful explanation and prediction algorithms for the mobility domain. This relies both on the efficient access to large amounts of integrated data described above and on the mobility expertise of our data science team. Learning models allow apps to not only rely on current and historical data but also use predictions about the future, for example planning a delivery route based on the likely traffic situation tomorrow at 5pm or recommending a parking space for Friday's commute. We can integrate and parallelize your existing techniques and models into the platform - this is especially easy if they're already coded in Python, Java or Scala.

Routing

We provide an extended version of the GraphHopper directions API (routing engine) for routing requests. This makes it possible not only to plan the quickest routes in general but also dependent on current or future traffic information for the network provided from data sources and predictions.
