SMARTLANE APIs

Introduction

This page details HTTP API calls you can make to the SMARTLANE platform. How to make API calls using your favorite language/framework is beyond the scope of this page, but we have good experience and expertise with shell commands, Python, Java and Javascript, so if you need help you can always ask!

Dataset management and Datastore API (CKAN API)

smartlane.io supports almost all API calls supported by CKAN. You can find full details in the CKAN API documentation and the CKAN Datastore API documentation.

API calls are made using action-named endpoints and should be POSTed with a JSON body when you wish to push new (meta)data to the platform. You can do this with your favorite basic HTTP requests methods but it's good to know that there higher level CKAN API libraries/modules available for Python, Java and Javascript/NodeJS. Don't forget to use https://smartlane.io as the server and your API key from your user management page (e.g. https://smartlane.io/user/edit/joebl) for authorization.

Augmentation API

This allows you to change GeoJSON properties using an additional web-source/API, facilitating for example realtime visualizations.

Endpoint:   https://smartlane.io/smartlaneweb/projects/DATASET_ID/resource/RESOURCE_ID/FEATURE_PROPERTY/augment (GET)
Parameters:
  lookupby:    property in the GeoJSON feature who's value should be used as a key in the augmentation file
  augmentedby: source of the style values for augmentation

For example, to set the fill color of features via the "Signal Group" GeoJSON property value from a URL:

https://smartlane.io/smartlaneweb/projects/9eb73fb6-4a5b-4ec8-8ec6-15da677cb23d/resource/0eabedf4-13e8-483b-b27f-3e7666a4be02/fill/augment?lookupby=Signal%20Group&augmentedby=https://smartlane.io/smartlaneweb/extern/frankfurt/lsa&apikey=API_KEY_IF_ACCESSING_PRIVATE_DATA
