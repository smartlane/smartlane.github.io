API Description

This document contains a detailed description of all API endpoints and respective data objects for SmartlaneDISPATCH! company, driver, address delivery management as well as route planning and optimisation, ETA calculation, accounting and user authorization. ## Getting started In order to use the API, you must be registered with SmartlaneDISPATCH! and an application instance must be assigned to your registered organization. You can request a competely free and unrestricted 30 day test version of the application via http://freetrial.smartlane.de/ or just write us at [info@smartlane.de](mailto:info@smartlane.de?subject=I'd like to try the SmartlaneDISPATCH API). We will then soon provide you a link to a registration form. Afterwards, your email address will be your user name and you will use a self-choosen password for accessing the API. The basic URL for all API calls (except /auth) is https://dispatch.smartlane.io/[your_organizations_name]/api Example call: GET https://dispatch.smartlane.io/smartlane/api/route/5 would retrieve all information for route nr. 5 from the organization Smartlane's database. There is a general limit of 20 API requests per second. If more requests are called, the response will be 429 Too Many Requests. If you need a higher request frequency, please et us know at support@smartlane.de. All our customers will automatically be supplied a web frontend visualizing all deliveries, routes and drivers. It is accessible in your web browser and using the same user credentials as for the API via https://dispatch.smartlane.io/[your_organizations_name]/ ## Authentication All API calls (besides some you need to access unauthorized like /api/resetpwd) are secured by JWT tokens. That means, you always need to provide an Authorization header with a token. The token is generated via the API call (without /api in this case) https://dispatch.smartlane.io/[your_organizations_name]/auth with a JSON body containing the following information: {'email': [your_registered_email_address], 'password' : [your_registered_password]} The result of this API call is a token which then can be used in all your API calls in the following way: HTTP header key: 'Authorization', header value: 'JWT [your_very_long_JWT_token]' (note the important prefix 'JWT '). The expiry time of a SmartlaneDISPATCH! JWT token is 1 day / 24 hours. That means, you have to call the /auth endpoint once per day and renew the token in all your calls respectively. ## Data privacy and security All API calls are secured by SSL hia HTTPS, so you don't have to worry about anybody intercepting your data via Internet transfers. All supplied data are stored securely on professionaly operated servers in Germany and they are only accessible by the API itself but are not exposed to direct access by anyone outside the application. The API itself is secured by a username and password which is supplied only to you in your registration process. Of course, passwords are always stored as encrypted hashes and thus are never readable by anybody. ## Detailed documentation of all API endpoints The following table contains a technical specification of all API endpoints usable for SmartlaneDISPATCH!

get /taskhistogram
Retrieves aggregated task data for billing. (taskhistogramGet)
Retrieves task entries aggregated by time intervals and task types with the following call. At the moment, the only task type considered is the creation of delivery objects, because only those are relevant for price calculation and API usage accounting.

Query parameters

type (optional)
Query Parameter — A string for the task type to group by (e.g. read or update). Default: all types (read, update, create, delete). At the moment, only create taks are collected in the database.
entity (optional)
Query Parameter — A string for the entity to group by (e.g. deliveries or routes). Default: all entities (see data model).At the moment, only delivery entities are collected in the database.
intervallength (optional)
Query Parameter — length of one aggregation interval in minutes (Default 5 minutes) format: int32
startdate (optional)
Query Parameter — Earliest start date of first interval. Format ISO 8601 (YYYY-MM-DDThh mm ss+tzh tzm) (Default 10 days back from now). timezone (+tzh tzm) is optional. Default ist local timezone format: dateTime
enddate (optional)
Query Parameter — latest end date of last interval. Format ISO 8601 (YYYY-MM-DDThh:mm:ss+tzh:tzm) (Default now). timezone (\"+tzh:tzm\") is optional. Default ist local timezone. format: dateTime
withnull (optional)
Query Parameter — When False (=Default) only intervals which contain any count value(s) from start date to end date will be returned by the API. When True all intervals from start date to end date will be returned. Count for intervals with no task is then always 0. Can be combined with all other parameters
